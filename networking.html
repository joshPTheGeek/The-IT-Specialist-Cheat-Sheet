Foundational Models: OSI vs. TCP/IP

To understand and troubleshoot network communication, IT professionals rely on conceptual frameworks that break the complex process into manageable layers. The two most important models are the Open Systems Interconnection (OSI) model and the TCP/IP model.

The OSI Model

A seven-layer framework that is protocol-agnostic, primarily used as a universal troubleshooting tool.

  1. Physical Layer: Transmission of raw bits over a medium (cables, fiber, radio).
  2. Data Link Layer: Node-to-node data transfer, physical addressing (MAC addresses), error detection.
  3. Network Layer: Packet forwarding, routing across networks, logical addressing (IP addresses).
  4. Transport Layer: Reliable end-to-end communication, flow control, segmentation (TCP/UDP).
  5. Session Layer: Manages, establishes, and terminates connections between applications.
  6. Presentation Layer: Translates data, handles encryption, compression, and formatting.
  7. Application Layer: Provides network services directly to software applications (HTTP, SMTP, DNS).

The TCP/IP Model

A more practical, four-layer model based on the actual protocol suite used for the internet.

  1. Network Access Layer: Combines OSI Layers 1 and 2.
  2. Internet Layer: Corresponds to OSI Layer 3.
  3. Transport Layer: Corresponds to OSI Layer 4.
  4. Application Layer: Combines OSI Layers 5, 6, and 7.

The mechanism that allows these models to function is **data encapsulation**. As data moves down the stack, each layer adds its own header (e.g., a TCP header creates a "segment," an IP header creates a "packet," a MAC header creates a "frame"). The receiving computer reverses this process (de-encapsulation).

Layer 2: The Data Link Layer

MAC Addresses: The Unique Hardware Identifier

A Media Access Control (MAC) address is a unique 48-bit identifier burned into a Network Interface Controller (NIC). It's used to deliver frames to the correct physical destination on a local network. The first 24 bits are the Organizationally Unique Identifier (OUI), identifying the manufacturer, and the last 24 bits are unique to the device.

Layer 2 Switching

A network switch operates at Layer 2, intelligently forwarding Ethernet frames using its **MAC address table**. The process involves four steps:

  1. Learning: The switch examines the source MAC address of an incoming frame and adds it to its table, associating it with the ingress port.
  2. Forwarding/Filtering: The switch looks up the destination MAC in its table. If found, it forwards the frame *only* to the corresponding port.
  3. Flooding: If the destination MAC is unknown (or is a broadcast), the switch sends the frame out of all ports except the one it came in on.

Layer 3: The Network Layer

IPv4 vs. IPv6 Addressing

IPv4: A 32-bit address (e.g., 192.168.1.100), allowing for ~4.3 billion addresses. Space exhaustion led to the creation of private IP ranges (RFC1918) and Network Address Translation (NAT).

IPv6: A 128-bit address (e.g., 2001:0db8::8a2e:0370:7334), providing a virtually inexhaustible number of addresses. It includes features like Stateless Address Autoconfiguration (SLAAC) and mandatory IPsec for security.

The Role of the Router

A router is a Layer 3 device that connects different networks or subnets. It uses a **routing table** to make decisions based on destination IP addresses, forwarding packets along the best path toward their final destination.

IP Subnetting

Subnetting is the process of dividing a single large network into multiple smaller logical networks (subnets). This improves efficiency, performance (by reducing broadcast domains), and security.

The Subnet Mask and CIDR

A **subnet mask** (e.g., 255.255.255.0) separates an IP address into a network portion and a host portion. **CIDR notation** (e.g., /24) is a more concise way to represent the mask, indicating the number of network bits.

Variable Length Subnet Masking (VLSM)

VLSM allows administrators to use different subnet masks for different subnets within the same network block. This "subnetting a subnet" technique enables precise allocation of IP addresses based on the specific needs of each network segment, virtually eliminating waste.

Virtual LANs (VLANs)

A VLAN allows network administrators to create logically separate networks on the same physical infrastructure. Devices are grouped into virtual networks, regardless of their physical location, creating separate broadcast domains.

Access vs. Trunk Ports

  • Access Port: Belongs to a single VLAN and connects to end devices (PCs, printers).
  • Trunk Port: Carries traffic for multiple VLANs simultaneously, typically connecting switches to other switches or routers. Uses the **IEEE 802.1Q** tagging standard.

Inter-VLAN Routing

Since VLANs are isolated, a Layer 3 device is required for them to communicate. Modern networks use a **Layer 3 Switch** with **Switched Virtual Interfaces (SVIs)**. An SVI is a virtual Layer 3 interface on the switch that acts as the default gateway for its assigned VLAN, enabling high-speed routing between VLANs.

The TCP Three-Way Handshake

The process TCP uses to establish a reliable connection:

  1. SYN: Client sends a synchronize request to the server.
  2. SYN-ACK: Server acknowledges the request and sends its own synchronize request back.
  3. ACK: Client acknowledges the server's request, and the connection is established.

Core Network Services

DNS: The Internet's Phonebook

The Domain Name System (DNS) translates human-friendly domain names (e.g., www.google.com) into the numerical IP addresses that computers use. This is a hierarchical process involving recursive resolvers, root servers, TLD servers, and authoritative name servers.

DHCP: Dynamic Host Configuration

The Dynamic Host Configuration Protocol (DHCP) automates the assignment of IP addresses and other network settings (subnet mask, gateway, DNS servers) to devices. It uses a four-step process known as **DORA**:

  1. Discover: Client broadcasts a request for an IP address.
  2. Offer: DHCP server(s) offer an available IP address.
  3. Request: Client formally requests one of the offered IPs.
  4. Acknowledge: Server confirms the lease and finalizes the configuration.

Wireless Networking

Wi-Fi Standards Evolution

The transition from **Wi-Fi 5 (802.11ac)** to **Wi-Fi 6 (802.11ax)** represents a significant leap, especially for dense environments. The key innovation in Wi-Fi 6 is **OFDMA**, which allows a single channel to be subdivided to communicate with multiple devices simultaneously, reducing latency and improving efficiency.
